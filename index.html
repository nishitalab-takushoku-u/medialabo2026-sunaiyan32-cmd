「Webコンテンツ作成」の成果

HTML・CSS・JavaScript を使って、情報を整理しながら見やすい Web ページを作る力を身につけることを目標に学習しています。 このページでは、第 1 回から第 6 回までに学んだ内容と、演習・課題として制作したページへのリンクをまとめています。

本テーマの学習目的

本テーマの目的は、HTMLで情報の構造を作り、CSSで見た目を整え、JavaScriptでページに動きを加える一連の流れを身につけることである。 Webページを単なる文章ではなく、利用者が読みやすく操作しやすい情報メディアとして設計する力を養う。

第 1 回 WWW と HTML

第 1 回では、Web ページが Web サーバからブラウザへ送られて表示される流れを学びました。 また、見出し・段落・箇条書き・リンク・画像などの HTML 要素を使い分け、内容に合った文書構造を作ることの大切さを理解しました。 単に文字を並べるのではなく、「どの情報が見出しで、どこが本文なのか」を意識してタグを選ぶことが、読みやすいページ作成の基本だと分かりました。

演習 1-4: HTML のさまざまな要素

第 2 回 CSS による装飾

第 2 回では、HTML で作った文書に CSS を適用して、文字の色・大きさ・余白・背景色・配置を調整する方法を学びました。 特に、外部スタイルシートを用いて複数ページに共通したデザインを適用することで、サイト全体の統一感を出せることを実感しました。 また、グリッドレイアウトを使うと、複数の情報を整理して並べやすくなり、見た目と可読性の両方を改善できると分かりました。

演習 2-3: browser-intro.html

第 3 回 JavaScript の基礎

第 3 回では、外部 JavaScript ファイルを HTML から読み込む方法や、コンソールを使って結果を確認する方法を学びました。 さらに、オブジェクトや配列から必要なデータを取り出して表示する練習を通して、ページに表示したい情報をプログラムで整理する考え方を身につけました。 project.html でも、天気データから必要な項目だけを選んで表示することを意識しています。

演習 3-6: obj-practice.html

第 4 回 DOM とイベント処理

第 4 回では、ボタンをクリックしたときに JavaScript を動かし、ページ内の要素を書き換える DOM 操作とイベント処理を学びました。 数当てゲームでは、テキストボックスに入力された予想を読み取り、回答ボタンを押した回数を数え、正解・不正解やヒントを画面に表示する処理を実装しました。 querySelector で要素を取得し、textContent を使って表示内容を更新することで、ページを動的に変化させられることが理解できました。 また、条件分岐を使って「正解」「もっと大きい」「もっと小さい」「ゲーム終了」といった結果を出し分けることで、入力に応じて動作を変える基本も練習できました。

演習 4-10: practice-eventdom/tashizan.html

第 5 回 DOM の応用

第 5 回では、JavaScript で新しい要素を作成し、既存のページへ追加したり、画像の属性を変更したりする練習をしました。 DOM の階層構造を意識して、見出しの後ろに段落やリストを挿入することで、データをページ上に整理して表示できるようになりました。 キャンパス情報や学科一覧をオブジェクト・配列から取り出し、ボタン操作で表示する処理も実装しました。

第 6 回 Ajax 通信

第 6 回では、Axios を使ってサーバから JSON データを取得し、通信に成功したとき・失敗したとき・最後に必ず実行する処理を分けて書く方法を学びました。 取得したデータが文字列の場合はオブジェクトに変換し、必要な値を取り出してコンソールやページに表示する流れを確認しました。 天気データの処理では、都市名、気温、湿度、風速などを JavaScript で読み取り、Web ページの表示へ反映できるようにしました。

プロジェクト課題: 世界の天気

プロジェクト課題について

プロジェクト課題では、「世界の天気」をテーマに選び、都市ごとの天気、気温、湿度、風の情報を確認できるページを制作しました。 表やリストを使って情報を整理し、必要な項目を素早く見つけられるように配置しました。 CSS ではカード型のレイアウトと余白を意識し、読みやすく落ち着いた見た目になるよう工夫しています。 JavaScript では天気データのオブジェクトから都市名、天気、最低気温、最高気温、湿度、経度、緯度、風速、風向を取り出し、対応する要素へ表示しました。 数値と説明文を分けて配置することで、利用者が一目で天気の概要を確認できるようにしました。 また、プロジェクトページ全体を成果ページと同じデザインにそろえ、課題として作成したページ同士の統一感も意識しました。

数当てゲームについて

数当てゲームでは、1から10までの乱数を正解として作り、利用者が入力した予想と比較する処理を実装しました。 回答ボタンを押すたびに予想回数を増やし、入力値が正解と同じ場合は「正解」、小さい場合は「もっと大きい」、大きい場合は「もっと小さい」と表示します。 3回以内に正解できなかった場合は答えを表示してゲームを終了し、終了後にさらにボタンを押しても結果が変わらないようにしました。 この課題を通して、乱数、変数、条件分岐、DOM の取得、イベント処理を組み合わせる練習ができました。 とくに、画面上の入力欄から値を読み取り、判定結果を同じページに表示する流れを自分で作れた点が重要でした。 JavaScript がページの内容を動的に変える仕組みを理解するきっかけになりました。

工夫点，苦心点，強調点，感想

今回の制作で最も工夫した点は、学習内容をただ並べるだけでなく、成果ページ全体を一つのポートフォリオとして見られるようにしたことである。 第1回から第6回までの内容はそれぞれ別の演習であるが、最終的に一つのページから確認できるようにリンクを整理し、どの回で何を学んだのかが分かる構成にした。 特に、プロジェクト課題、数当てゲーム、各回の演習ページへのリンクを分かりやすいボタンとして配置し、閲覧者が迷わず目的のページへ移動できるようにした。 また、最初は文章量が少なく、課題で求められている説明としては不十分だったため、それぞれの課題で何を作り、どのような技術を使い、何を理解したのかを具体的に書くように修正した。

苦心した点は、HTML、CSS、JavaScript の役割を分けて考えることである。 HTML では見出しや段落、表、リスト、リンクなどを使って情報の構造を作る必要がある。 CSS では色や余白、配置を調整するが、見た目だけを優先すると読みにくくなるため、文字の大きさや行間、カード同士の間隔を意識した。 JavaScript では、どの要素を取得し、どのタイミングで処理を実行し、どの内容を表示するのかを順番に考える必要があった。 特に DOM 操作では、要素を新しく作る、属性を変える、テキストを書き換える、イベントを登録するという処理が混ざるため、最初はどこで何が起きているのか分かりにくかった。 しかし、処理を関数に分け、ボタンを押したときに実行される流れを確認することで、少しずつ理解できるようになった。

強調した点は、完成したページが実際に使える状態になっていることである。 数当てゲームでは、入力、判定、回数表示、終了処理が画面上で確認できるようにした。 Ajax 通信のデモでは、もともと結果がコンソールにしか出なかったため、ページ上にも通信結果を表示し、ボタンを押したことが利用者に伝わるようにした。 プロジェクト課題では、天気データを表とリストに分けて表示し、都市名や気温などの重要な情報を見つけやすくした。 デザイン面では、最近の中国の大手IT企業やクラウドサービスのページに見られる、暗い背景、細いグリッド、青い光、赤いアクセントを参考にして、より現代的で技術感のある見た目にした。 ただし、装飾が強すぎると課題内容が読みにくくなるため、カードの背景、文字色、余白を調整し、本文の読みやすさを優先した。

感想として、Webページ制作は一つの技術だけで完成するものではなく、構造、見た目、動き、情報整理を同時に考える必要があると感じた。 HTML のタグを正しく使わないと情報の意味が伝わりにくく、CSS がないと見やすいページにならず、JavaScript がないと利用者の操作に反応できない。 それぞれの技術は別々に学ぶが、実際の制作では互いにつながっていることが分かった。 また、文字化けやリンク切れ、ボタンを押しても見た目が変わらない問題など、細かい部分でも利用者にとっては大きな違和感になることを実感した。 今後は、見た目を整えるだけでなく、スマートフォンでの表示、アクセシビリティ、データの取得方法、エラー時の表示なども意識して、より実用的なページを作れるようになりたい。 今回の課題を通して、自分で作ったページを何度も確認し、問題を見つけて修正することの大切さを学んだ。
