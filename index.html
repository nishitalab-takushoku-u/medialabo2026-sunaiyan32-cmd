「Webコンテンツ作成」の成果

HTML・CSS・JavaScript を使って、情報を整理しながら見やすい Web ページを作る力を身につけることを目標に学習しています。 このページでは、第 1 回から第 4 回までに学んだ内容と、演習・課題として制作したページへのリンクをまとめています。

project.html 数当てゲーム たし算ページ 演習 2-3

第 1 回 WWW と HTML

第 1 回では、Web ページが Web サーバからブラウザへ送られて表示される流れを学びました。 また、見出し・段落・箇条書き・リンク・画像などの HTML 要素を使い分け、内容に合った文書構造を作ることの大切さを理解しました。 単に文字を並べるのではなく、「どの情報が見出しで、どこが本文なのか」を意識してタグを選ぶことが、読みやすいページ作成の基本だと分かりました。

演習 1-4: HTML のさまざまな要素

第 2 回 CSS による装飾

第 2 回では、HTML で作った文書に CSS を適用して、文字の色・大きさ・余白・背景色・配置を調整する方法を学びました。 特に、外部スタイルシートを用いて複数ページに共通したデザインを適用することで、サイト全体の統一感を出せることを実感しました。 また、グリッドレイアウトを使うと、複数の情報を整理して並べやすくなり、見た目と可読性の両方を改善できると分かりました。

演習 2-3: browser-intro.html

第 3 回 JavaScript の基礎

第 3 回では、外部 JavaScript ファイルを HTML から読み込む方法や、コンソールを使って結果を確認する方法を学びました。 さらに、オブジェクトや配列から必要なデータを取り出して表示する練習を通して、ページに表示したい情報をプログラムで整理する考え方を身につけました。 project.html でも、天気データから必要な項目だけを選んで表示することを意識しています。

演習 3-6: obj-practice.html

第 4 回 DOM とイベント処理

第 4 回では、ボタンをクリックしたときに JavaScript を動かし、ページ内の要素を書き換える DOM 操作とイベント処理を学びました。 数当てゲームでは、テキストボックスに入力された予想を読み取り、回答ボタンを押した回数を数え、正解・不正解やヒントを画面に表示する処理を実装しました。 querySelector で要素を取得し、textContent を使って表示内容を更新することで、ページを動的に変化させられることが理解できました。 また、条件分岐を使って「正解」「もっと大きい」「もっと小さい」「ゲーム終了」といった結果を出し分けることで、入力に応じて動作を変える基本も練習できました。

課題: 数当てゲーム

演習 4-10: practice-eventdom/tashizan.html

プロジェクト課題について

プロジェクト課題では、「世界の天気」をテーマに選び、都市ごとの天気、気温、湿度、風の情報を確認できるページを制作しました。 表やリストを使って情報を整理し、必要な項目を素早く見つけられるように配置しました。 CSS ではカード型のレイアウトと余白を意識し、読みやすく落ち着いた見た目になるよう工夫しています。
